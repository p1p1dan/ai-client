## Task: Add onboarding flow for automated registration and AI tool configuration

Implement the complete onboarding system in the Electron client.

### 1. Add IPC channels in `src/shared/types/ipc.ts`

Add before the `// Logging` section in the IPC_CHANNELS object:
```typescript
  // Onboarding
  ONBOARDING_CHECK: 'onboarding:check',
  ONBOARDING_REGISTER: 'onboarding:register',
  ONBOARDING_DETECT_CLI: 'onboarding:detectCli',
```

### 2. Create `src/shared/types/onboarding.ts`

Define these interfaces:
- OnboardingState: { registered: boolean; email?: string; serverUrl?: string; registeredAt?: string }
- OnboardingRegisterRequest: { email: string; serverUrl: string; onboardingSecret: string }
- OnboardingRegisterResponse: { ok: boolean; error?: string; data?: { user: { id: number; name: string }; apiKey: string; config: { claude: { baseUrl: string; authToken: string }; codex: { baseUrl: string; apiKey: string } } } }
- OnboardingCliStatus: { claudeInstalled: boolean; claudeVersion?: string; codexInstalled: boolean; codexVersion?: string }

Export from `src/shared/types/index.ts`.

### 3. Create `src/main/services/onboarding/OnboardingService.ts`

A service class with:

- `checkRegistration()`: Reads onboarding field from ~/.ensoai/settings.json, returns OnboardingState
- `validateEmail(email)`: Validates format and @jcdz.cc suffix
- `register(email, serverUrl, onboardingSecret)`:
  - Uses `net.fetch()` from Electron to POST to `${serverUrl}/api/onboarding/register`
  - Sends X-Onboarding-Secret header
  - On success, applies Claude config via existing `applyProvider()` from `../claude/ClaudeProviderManager`
  - Also writes Codex config to ~/.codex/env.json (OPENAI_API_KEY and OPENAI_BASE_URL)
  - Saves onboarding state to ~/.ensoai/settings.json (read-modify-write the settings file)
  - Returns OnboardingRegisterResponse
- `detectCli()`: Uses existing `cliDetector.detectOne('claude')` and `cliDetector.detectOne('codex')` to check CLI status

Export as singleton: `export const onboardingService = new OnboardingService();`

### 4. Create `src/main/ipc/onboarding.ts`

Register three IPC handlers:
- ONBOARDING_CHECK -> onboardingService.checkRegistration()
- ONBOARDING_REGISTER -> onboardingService.register(request.email, request.serverUrl, request.onboardingSecret)
- ONBOARDING_DETECT_CLI -> onboardingService.detectCli()

Export function `registerOnboardingHandlers()`.

### 5. Modify `src/main/ipc/index.ts`

Add import and call `registerOnboardingHandlers()` in `registerIpcHandlers()`.

### 6. Add to preload bridge in `src/preload/index.ts`

Add `onboarding` namespace to the electronAPI object with three methods: check, register, detectCli.
Each calls ipcRenderer.invoke with the corresponding IPC channel.

Import OnboardingState, OnboardingRegisterRequest, OnboardingRegisterResponse, OnboardingCliStatus types.

### 7. Create `src/renderer/components/onboarding/OnboardingDialog.tsx`

A React modal dialog with 3 steps:

Step 1 - CLI Check:
- Shows Claude CLI and Codex CLI installation status
- Green check for installed, warning for not installed
- "Next" button to proceed (even if CLI not installed - warn but allow)

Step 2 - Registration:
- Server URL input (with a default, stored in settings)
- Onboarding Secret input (password field, stored in settings)
- Email input with real-time @jcdz.cc validation
- "Register" button that calls window.electronAPI.onboarding.register()
- Loading state during registration
- Error display with retry

Step 3 - Success:
- Shows success message with configured tools
- "Done" button that closes dialog

Props: `{ open: boolean; onComplete: () => void }`

Use existing UI components from src/renderer/components/ui/ - look for Dialog, Button, Input components.
Use Lucide React icons (CheckCircle2, AlertCircle, Loader2, Mail, Server, Key).
Follow the design system: CSS variables for colors (text-primary, bg-accent, etc).

### 8. Integrate in `src/renderer/App.tsx`

Add at the top of the App component:
- State: `const [showOnboarding, setShowOnboarding] = useState(false);`
- useEffect that calls `window.electronAPI.onboarding.check()` on mount
- If not registered, set showOnboarding to true
- Render OnboardingDialog conditionally
